1   Introduzione
Il linguaggio TecSQL permette di interrogare e manipolare i dati memorizzati in un database relazionale supportato da Tecnest (attualmente Oracle, Informix, SQLite, FastDB e MySQL). La sua sintassi rimane inalterata a prescindere dall'effettivo motore db utilizzato, grazie ad un processo di "parsing" che traduce a runtime le istruzioni TecSQL nel linguaggio compatibile con il database in uso.  TecSQL è fortemente legato ai Descrittori di Tabella, in quanto può utilizzare i nomi logici delle tabelle e dei campi così come definiti in tali descrittori.  TecSQL è utilizzato internamente nel codice sorgente di JFlex, ma viene sfruttato anche da alcuni tool utilizzabili dall'utente finale per creare Interrogazioni, Info contestuali e Report. 
    
2   Sintassi TecSQL
Si illustrano ora alcune convenzioni addottate nella definizione della sintassi TecSQL riportata in seguito:  • tutte le espressioni in "minuscolo" sono espressioni sintattiche non terminali, quindi ulteriormente riducibili  • tutte le espressioni in "MAIUSCOLO" sono parole chiave  • "table_name" e "column_name" sono nomi che identificano tabelle o campi di tabella; nel caso vengano preceduti dal simbolo $ (dollaro) vengono considerati nomi logici così come definiti nei Descrittori di Tabella; in caso contrario vengono considerati nomi fisici. "alias" introduce una abbreviazione che può essere utilizzata al posto di un nome di tabella o di campo. Esempi:    $Articolo - nome logico di tabella   $Codice - nome logico di campo   $Articolo.Codice - nome logico di campo che specifica anche il nome logico della tabella a cui appartiene   MD_ARTI - nome fisico di tabella   ARTI_ARTCO - nome fisico di campo   MD_ARTI.ARTI_ARTCO - nome fisico di campo che specifica anche il nome fisico della tabella a cui appartiene   T1.Codice - se T1 è un alias di un nome logico allora Codice è un nome logico, altrimenti è un nome fisico  • "string" è una stringa SQL, come ad esempio 'stringa di prova'  • "date" è una data in formato yyyy-MM-dd  • "time" è un istante in formato hh:mm:ss  • "timestamp" è un istante in formato yyyy-MM-dd hh:mm:ss  • "number" è un numero intero o decimale che utilizza il . (punto) come separatore decimale

2.2   INSERT
insert_statement ::=         INSERT INTO table_name [ ( column_name [, column_name]* ) ] values  values ::=         expression [, expression]* | ( select ) 
 
2.3   DELETE
delete_statement ::=         DELETE FROM table_name [ where_clause ] 
 
2.4   UPDATE
update_statement ::=         UPDATE table_name SET set _element [, set_element ]* [ where_clause ]  set_element ::=         column_name = expression 
 
2.5   LOCK
lock_statement ::=         LOCK TABLE table_name IN EXCLUSIVE MODE |         LOCK TABLE table_name IN SHARE MODE 
    
3   Funzioni
• TRIM(p1 [, p2]) - elimina i caratteri blank in testa ed in coda al valore stringa specificato in p1     p1: la stringa da trimmare     p2: l'eventuale carattere da eliminare diverso dal blank     return: la stringa trimmata  • RTRIM(p1 [, p2]) - elimina i caratteri blank in coda al valore stringa specificato in p1     p1: la stringa da trimmare     p2: l'eventuale carattere da eliminare diverso dal blank     return: la stringa trimmata  • LTRIM(p1 [, p2]) - elimina i caratteri blank in testa al valore stringa specificato in p1     p1: la stringa da trimmare     p2: l'eventuale carattere da eliminare diverso dal blank     return: la stringa trimmata  • LPAD(p1, p2 [, p3]) - riempie la testa della stringa p1 con un carattere di riempimento (blank) in modo che assuma la lunghezza definita in p2     p1: la stringa da riempire     p2: la lunghezza della stringa che si vuole ottenere     p3: l'eventuale carattere di riempimento diverso dal blank     return: la stringa riempita  • RPAD(p1, p2 [, p3]) - riempie la coda della stringa p1 con un carattere di riempimento (blank) in modo che assuma la lunghezza definita in p2     p1: la stringa da riempire     p2: la lunghezza della stringa che si vuole ottenere     p3: l'eventuale carattere di riempimento diverso dal blank     return: la stringa riempita  • UPPER(p1) - 4   Limitazioni
 
4.1   Limitazioni FastDB
Il database FastDB ha grosse limitazioni rispetto ai comandi SQL che possono essere eseguiti su un DB tradizionale. In una sessione FastDB non è quindi possibile usare:  • le sub-select • le UNION • i costrutti GROUP BY, HAVING e FOR UPDATE • le join • il costrutto DISTINCT • il costrutto table_name.* • l'alias di tabella e di campo • le funzioni di aggregazione • lo statement SELECT in operazioni di INSERT • le funzioni: ABS, COS, DATE, LENGTH, LOG, LOGN, LOWER, MOD, POW, ROUND • le funzioni: SIN, SQRT, TODAY, TAN, TRIM, TRUNC, UPPER, YEAR, WEEK, DAY, MONTH • i parametri e gli operatori variabili • lo statement CASE WHEN 
 
4.2   Limitazioni SQLite
Sebbene il linguaggio SQLite supporti quasi completamente le potenzialità del linguaggio SQL standard, vi sono alcune limitazioni. In una sessione SQLite non è possibile usare:  • i RIGHT JOIN • i costrutti FOR UPDATE • le sub-select in espressioni • la funzione PAD  ATTENZIONE: al momento la funzione WEEK ha una diversa implementazione in SQLite rispetto ad Oracle, il risultato può quindi essere diverso. 
    
5   Estensioni speciali
La sintassi del linguaggio TecSQL è stata estesa per supportare alcune funzionalità speciali fruibili in alcuni contesti particolari, quali:    • le interrogazioni (query)   • le info contestuali   • i report (Jasper) 
 
5.1   Parametri
I parametri sono particolari elementi di una query il cui valore viene assegnato a runtime. Normalmente il valore dei parametri viene inserito dall'utente tramite apposite finestre di dialogo costruite in automatico dal sistema. I parametri delle info invece prevedono il recupero automatico del valore dal contesto chiamante.  La sintassi è la seguente:  <expression> <operator> <parameter> | <parameter> <operator> <expression>  <expression> è una qualsiasi espressione. Se <expression> corrisponde al nome logico di un campo di tabella, il sistema è in grado di recuperare dal descrittore le caratteristiche del campo (label, font, colori, ecc.) e quindi di configurare a dovere il campo sulla finestra di dialogo di richiesta parametri. Altrimenti verrà mostrato un campo generico coerente con il tipo dell'espessione.  <operator> è un qualsiasi operatore di confronto supportato da TecSQL, oppure l'indicatore di operatore variabile (vedi paragrafo successivo).  <parameter> ::= ?<name> | ?!<name>  <name> è il nome del parametro  Esempio:    $Articolo.Codice = ?CodArticolo  Il campo presente in questa espressione booleana ($Articolo.Codice) permette al sistema di ricavare tutte le informazioni necessarie alla richiesta di input del parametro (tramite il descrittore).  I parametri devono avere come prefisso il carattere ? (punto interrogativo). Il parametri con prefisso ?! (punto interrogativo seguito da punto esclamativo) vengono considerati parametri obbligatori. I parametri non valorizzati vengono esclusi dalla query assieme all'espressione a cui sono legati. 
 
5.2   Operatori variabili
Utilizzando la speciale notazione di "operatore variabile" è possibile costruire query con parametri in cui l'operatore verrà scelto a runtime dall'utente. L'operatore variabile viene identificato dal carattere # (cancelletto). Esempio:    $Articolo.Codice # ?CodArticolo  E' possibile assegnare un default all'operatore variabile facendo seguire al # l'operatore inizialmente presentato all'utente:    #= operatore variabile con default = (uguale)   #> operatore variabile con default > (maggiore)   #>= operatore variabile con default >= (maggiore uguale)   #<= operatore variabile con default <= (minore uguale)   #< operatore variabile con default < (minore)   #!= operatore variabile con default != (diverso)   #BETWEEN operatore variabile con default [] (da/a) 
    
6   Esclusione del parser TecSQL
E' possibile inviare al db-server istruzioni non supportate dalla sintassi TecSQL facendole precedere dal suffisso "AS IS", che fa in modo di disattivare il controllo sintattico del parser TecSQL. Esempio:     AS IS ALTER SESSION SET NLS_SORT = 'BINARY'  viene eseguita come    ALTER SESSION SET NLS_SORT = 'BINARY'
